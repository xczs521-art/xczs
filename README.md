# 照章成事游戏工坊 · 后端服务说明

技术栈：**Node.js + Express + SQLite**（数据库用的是 Node.js 22+ 内置的 `node:sqlite` 模块，不需要额外安装数据库软件，也不需要本地编译原生模块）

---

## 目录结构

```
backend/
├── server.js          后端主程序（API + 联网对战 Socket.IO 逻辑全部在这一个文件里）
├── package.json        依赖清单
├── start-game.bat         【小白友好】双击就能自动装依赖+启动+开浏览器，发给朋友用这个
├── start-public.bat    进阶：配合 cloudflared 把本机服务临时暴露到公网
├── arcade.db            数据库文件（首次启动自动生成，不用手动建）
└── public/
    └── index.html       前端页面（就是游戏工坊本体，含井字棋/五子棋/扫雷三个游戏）
```

## 🚀 把整个文件夹发给朋友，对方要怎么打开运行？

**把 `backend` 这整个文件夹（或者打包成的 zip）发给对方，对方收到后：**

1. 如果是 zip，先解压到任意目录（比如桌面）
2. **确认电脑装了 Node.js**（版本 ≥ 22.5）。没装的话，去 https://nodejs.org 下载 "LTS" 版本装一下，一路下一步就行，装好后可能需要重启一下电脑
3. **双击文件夹里的 `start-game.bat`**
   - 第一次运行会自动装依赖（等它跑完，可能要几十秒），之后每次启动都是秒开
   - 会自动弹出一个后端服务窗口（黑色命令行，**开着别关**，关了游戏就用不了了）
   - 会自动打开浏览器访问 http://localhost:3001 ，看到游戏工坊首页就说明成功了

对方全程不需要懂任何命令行知识，只要双击这一个 `.bat` 文件即可。

**⚠️ 关于"能不能生成一个真正免装 Node 的 .exe"：**
说实话直接说明一下限制：这个项目的数据库用的是 Node.js 22.5+ 才有的内置 `node:sqlite` 模块，市面上常见的把 Node 项目打包成单文件 exe 的工具（比如 pkg、nexe）目前捆绑的都是比较旧的 Node 运行时，普遍不支持这个新模块，硬打包大概率运行时会报错。所以没有直接生成一个"双击就跑、完全不用装任何东西"的 exe。
`start-game.bat` 已经把"装依赖 + 开后端 + 开浏览器"这三步自动化成了一次双击，体验上已经很接近真正的 exe 了，额外只需要对方提前装一次 Node.js（装好之后什么工具项目都能用，不是一次性的）。
如果你确实需要连 Node 都不用装的纯 exe（比如要给完全不装开发工具的电脑用），可以告诉我，我可以帮你把数据库部分换成不依赖 `node:sqlite` 的实现，之后就能用标准工具打包成真正的单文件可执行程序了。

## 💻 怎么和朋友联机对战（跨设备/跨网络）

联网对战（井字棋 / 五子棋 / 扫雷 都支持）依赖 Socket.IO 实时通信，**前提是双方连的必须是同一个正在运行的后端服务**。如果你和朋友各自在自己电脑上分别双击启动，那是两个完全独立、互不相通的服务器，房间号在对方那边根本不存在。按需要选一种方案：

**方案一 · 同一个局域网（比如同一个 WiFi、同一个宿舍/教室）**
- 只需要一个人启动服务（双击 `start-game.bat`）
- 另一方不用启动任何东西，直接在浏览器里访问 `http://<启动服务那台电脑的局域网IP>:3001`（比如 `http://192.168.1.5:3001`），局域网 IP 可以在启动服务那台电脑上用 `ipconfig` 查看（找"IPv4 地址"那一行）
- 如果连不上，检查一下启动服务那台电脑的防火墙有没有拦截 3001 端口

**方案二 · 不在同一网络，想临时联机一次（推荐，最简单）**
- 用 `start-public.bat`（同目录下已经有这个脚本），它会用 [cloudflared](https://github.com/cloudflare/cloudflared) 工具把本机的服务临时映射出一个公网网址（类似 `https://xxxx.trycloudflare.com`）
- 第一次用需要先装 cloudflared：`winget install --id Cloudflare.cloudflared -e`
- 双击 `start-public.bat`，等它打印出 `trycloudflare.com` 的网址，把这个网址发给朋友，两人都打开这个网址玩就行
- 缺点：这个临时网址只在你的 `start-public.bat` 窗口开着的时候有效，关掉窗口网址就失效了，适合"现在就打一把"，不适合长期使用

**方案三 · 想要一个长期稳定、随时都能连的地址**
- 参考下面「部署到你自己的服务器」或「免费部署到 Render.com」两节，把后端部署到公网，双方直接访问这个固定网址

## 🆕 本次更新：扫雷加入联网对战

扫雷现在和井字棋、五子棋一样，多了「🌐 联网对战」模式：

- 创建房间的一方可以选择难度（8×8 / 12×12 / 14×14），加入房间的一方难度会自动跟创建者保持一致，不需要也不能自己选
- 双方看到的是**完全相同的一张雷区地图**（服务器生成一个随机种子下发给两边，两边各自用同一个种子还原出一样的雷区布局，不需要传输整张地图数据）
- 规则和同屏双人「轮盘赌」模式一致：两人轮流翻开格子，没有首点保护，谁先踩到雷谁当场落败，安全格子全部翻开则平局
- 点「重新开始」会让双方同步换一张全新的随机地图，不会出现两边地图不一致的情况
- 对方掉线会有提示；这一局的比分只在当前网页会话里统计，刷新页面或换房间会清零（不写入数据库排行榜，和同屏轮盘赌模式的记分规则保持一致）



- **Node.js 版本 ≥ 22.5**（因为用到了实验性的 `node:sqlite` 模块）
  检查方法：终端输入 `node -v`，如果低于 22.5，去 https://nodejs.org 下载最新 LTS 版本重装即可

## 安装 & 运行

```bash
cd backend
npm install       # 只装 express 和 cors 两个包，很快
npm start         # 或者 node server.js
```

看到这行说明启动成功：

```
照章成事游戏工坊后端已启动: http://localhost:3001
数据库文件: /xxx/backend/arcade.db
```

浏览器打开 **http://localhost:3001** 就能看到游戏工坊主页（前端页面是后端直接托管的，同源访问，不会有跨域问题）。

## 数据库说明

- 数据库文件是 `arcade.db`，SQLite 单文件数据库，首次启动自动创建表结构
- 想直接查看数据库内容的话，可以用 [DB Browser for SQLite](https://sqlitebrowser.org/)（免费图形化工具）打开 `arcade.db` 文件
- 表结构一共5张表：
  - `users` 用户账号（用户名 + 密码哈希）
  - `sessions` 登录会话（token）
  - `game_time` 每个用户累计的游戏时长
  - `duel_stats` 井字棋/五子棋 人机对战 的胜负平统计
  - `mine_best` 扫雷各难度的最佳用时

## API 接口一览

| 方法 | 路径 | 说明 | 需要登录 |
|---|---|---|---|
| POST | /api/register | 注册账号 `{username,password}` | 否 |
| POST | /api/login | 登录 `{username,password}` | 否 |
| POST | /api/logout | 登出 | 是 |
| GET | /api/me | 获取当前用户信息 | 是 |
| POST | /api/gametime | 上报游戏时长增量 `{delta}` | 是 |
| POST | /api/duel-result | 提交对战结果 `{game,result}` | 是 |
| GET | /api/leaderboard/duel/:game | 井字棋/五子棋排行榜 | 否 |
| POST | /api/mine-result | 提交扫雷成绩 `{difficulty,time}` | 是 |
| GET | /api/leaderboard/mine | 扫雷排行榜 | 否 |
| GET | /api/health | 健康检查 | 否 |

登录接口返回一个 `token`，前端把它存进 `localStorage`，之后每次请求都在请求头里带上：
`Authorization: Bearer <token>`

## 部署到你自己的服务器（VPS）

如果要放到公网让别人也能访问，比如你有 SSH 能登录的节点：

```bash
# 1. 把整个 backend 文件夹上传到服务器
scp -r backend/ user@你的服务器IP:/home/user/

# 2. 登录服务器
ssh user@你的服务器IP
cd backend
npm install

# 3. 用 pm2 常驻运行（推荐，断线不会停）
npm install -g pm2
pm2 start server.js --name arcade
pm2 save
pm2 startup   # 让它开机自启

# 4. 浏览器访问
http://你的服务器IP:3001
```

如果服务器防火墙没开 3001 端口，记得放行：`ufw allow 3001`（Ubuntu）或对应云服务商的安全组规则里加一条。

## 联网对战功能

井字棋、五子棋、扫雷现在都支持「🌐 联网对战」模式，用的是 Socket.IO 实时通信，房间号配对：

- 一方点「创建房间」，拿到一个 5 位房间号（比如 `C5K79`），发给对方
- 对方输入这个房间号，点「加入房间」，双方自动开始对战
- 创建房间的人先手，走子/翻格会实时同步到对方屏幕上，服务器会校验回合顺序（不是你的回合点了也没用，防止误触/作弊）
- 井字棋、五子棋的对局结果会各自记入自己账号的排行榜（如果登录了的话）；扫雷联网对战的胜负目前只在当页会话内计分，不写入数据库

**⚠️ 重要：联网对战只有双方连的是同一个正在运行的后端服务时才能用。**
如果你和对方各自在自己电脑上分别启动了一份 `node server.js`，那是两个完全独立、互不相通的服务器，房间号在对方那边根本不存在，自然连不上。要跨网络（比如你在学校、老师在家）对战，必须把这个后端**部署到一个双方都能访问到的公网地址**，然后两人都打开同一个网址来玩，而不是各自跑本地服务。

### 免费部署到公网（推荐：Render.com，5分钟搞定，不用信用卡）

1. 把 `backend` 这个文件夹整个上传到一个新的 GitHub 仓库（在 github.com 上新建仓库 → 上传文件，或用 `git push`）
2. 打开 https://render.com ，用 GitHub 账号登录
3. 点 **New +** → **Web Service**，选择你刚才那个仓库
4. 配置：
   - **Build Command**: `npm install`
   - **Start Command**: `node server.js`
   - **Instance Type**: 选 Free
5. 点 Create Web Service，等它构建完成（一两分钟），会给你一个类似 `https://你的项目名.onrender.com` 的网址
6. 你和老师都打开这个网址，登录/注册各自的账号，进井字棋或五子棋选「联网对战」，就能互相创建/加入房间对战了

**注意**：Render 免费套餐的实例在没人访问 15 分钟后会休眠，下次访问需要等 30~50 秒"冷启动"，属于正常现象，对战前提前把网址打开预热一下就行。另外免费套餐的磁盘不是持久化的，服务重启后 `arcade.db` 里的账号数据会清空——如果只是给老师演示对战功能，这完全没问题；如果想要账号数据长期保留，需要升级到付费套餐挂载持久磁盘，或换用外部数据库。



如果只是想临时看一下界面效果，`public/index.html` 也可以直接双击用浏览器打开（不需要开后端）。这种情况下它会自动尝试连接 `http://localhost:3001`，如果后端没开着，登录/注册会提示"连接不上后端服务"——这是正常的，说明前端的容错处理生效了，不是 bug。

## 安全性说明（诚实提醒）

这套账号系统是为了满足课程"接数据库、写后端"的要求而做的教学级实现，密码用的是 SHA-256 哈希（没加盐、没用 bcrypt 这类专业密码哈希算法），token 也是明文存在数据库里没有过期时间。**不建议存真实密码或用于生产环境**，但作为课程作业演示"前后端分离 + 数据库持久化"这个技术点是完全够用的。如果老师问起，可以如实说明这一点，也是一个可以在答辩里加分的"我们知道这里的局限性、如果做成真产品会怎么改进"的谈资。
